# Editorial Ledger

A full-page AgentSite archetype for credibility-first pages that should read like a calm operating memo, not a dark product dashboard.

## Classification

```yaml
kind: archetype
archetype: editorial-ledger
visual_preset: editorial-light
```

This is intentionally different from `copy-evidence-strip`, which is a section recipe. `editorial-ledger` owns the page skeleton and visual language.

## Visual intent

- light paper background
- serif editorial headline rhythm
- memo/ledger structure
- numbered entries
- claim-to-evidence rows
- visible boundaries without dashboard/status-panel language

## Use when

Use this when the brief asks for:

- trust, credibility, evidence, provenance, or claim discipline
- memo/briefing/editorial narrative
- artifact-backed service positioning
- a page that must visibly diverge from cockpit-dark SaaS styling

## Avoid

Do not use this as a fake case study, testimonial wall, or customer-proof page. It should cite repo artifacts, contracts, QA output, screenshots, plans, and user-provided facts only.
