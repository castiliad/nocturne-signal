# Primary Surface Layout acceptance

Before claiming this recipe is applied:

- [ ] The site records a primary surface id.
- [ ] The primary module appears before `#secondary-modules`.
- [ ] Secondary app-like modules are wrapped in keyboard-accessible `<details>` sections.
- [ ] Generic proof/boundary/CTA sections remain visible and reachable.
- [ ] `npm run check:layout` passes.
- [ ] `npm run qa:full` passes.
- [ ] Visual review confirms the primary task is no longer buried.
- [ ] No analytics, tracking, auth, live sync, persistence claims, server runtime, or database behavior were introduced.
